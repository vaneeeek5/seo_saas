ADR-002
Modular Monolith
Статус
Approved
Контекст
Выбрать архитектуру приложения.
Рассматривались:
	•	Monolith
	•	Modular Monolith
	•	Microservices
Решение
Использовать Modular Monolith.
Причины
На этапе MVP микросервисы создают лишнюю сложность.
Монолит плохо масштабируется.
Modular Monolith обеспечивает:
	•	единый деплой;
	•	простую локальную разработку;
	•	четкие границы модулей;
	•	возможность разделения на сервисы в будущем.
Последствия
Каждый модуль обязан иметь:
	•	собственный namespace;
	•	собственные сервисы;
	•	собственные модели;
	•	собственный API;
	•	собственные события;
	•	собственные тесты.
Запрещено использовать внутренние классы другого модуля.
