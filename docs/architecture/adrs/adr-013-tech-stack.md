# ADR-013: Technology Stack

## Статус
Принято

## Контекст
Платформе SEO Content Factory требуется надежный, масштабируемый и современный технологический стек. Нам необходимо поддерживать сложные асинхронные операции (AI генерация), модульную монолитную архитектуру, работу в реальном времени и хранение векторных данных для AI-агентов. Важно выбрать инструменты, которые обеспечат баланс между скоростью разработки, производительностью и возможностью дальнейшего масштабирования.

## Решение
Мы фиксируем следующий технологический стек для платформы:

- **Язык программирования:** TypeScript. Используется повсеместно (Frontend и Backend) для обеспечения строгой типизации и переиспользования моделей (Domain-Driven Design).
- **Backend Framework:** NestJS (допускается использование Express в легковесных модулях). NestJS предоставляет отличную структуру для реализации Modular Monolith и паттернов CQRS, описанных в архитектуре.
- **Frontend Framework:** React с использованием Next.js. Next.js обеспечит поддержку SSR/SSG, что полезно для SEO-интерфейсов и быстрой загрузки дашбордов.
- **Основная база данных:** Firebase / Firestore. NoSQL подход позволит гибко хранить документы проекта (Website, Semantic, Articles) без жестких миграций схемы на начальном этапе.
- **Очереди и асинхронные задачи:** Redis + BullMQ. Поскольку система основана на Event-Driven подходе и Task Engine, BullMQ обеспечит надежную доставку сообщений, обработку отложенных задач (Workers) и повторные попытки при сбоях AI-агентов.
- **Векторная база данных:** Pinecone или Qdrant. Необходима для работы Knowledge Engine, поиска контекста (RAG) и хранения эмбеддингов (Embeddings) семантического ядра и статей.

## Последствия
**Положительные:**
- Единый язык (TypeScript) для всей кодовой базы снижает порог входа и упрощает поддержку.
- Инфраструктура готова к сложным AI-вычислениям благодаря BullMQ и Vector DB.
- NestJS предоставляет встроенную поддержку микросервисной и модульной архитектуры, что полностью соответствует нашему принципу Modular Monolith.

**Отрицательные:**
- Разработчикам потребуется глубокое знание NestJS и реактивного программирования (RxJS) для реализации сложных Event-Driven паттернов.
- Использование Firebase может наложить ограничения на сложные реляционные запросы в будущем (потребуется дублирование данных или адаптация CQRS Read-моделей).
