# ADR-015: AI Orchestration & LLM Integration

## Статус
Принято

## Контекст
Платформа выполняет функцию автономного SEO-агента, что требует интенсивной работы с большими языковыми моделями (LLM). Для того чтобы система могла программно взаимодействовать с ответами от AI, необходимо обеспечить строгую типизацию вывода LLM. Кроме того, модуль Knowledge Engine должен обладать актуальным контекстом проекта для написания качественного контента.

## Решение
Устанавливаются следующие правила интеграции с LLM:

1. **Structured Outputs (JSON Schema):** Все запросы к LLM (за исключением прямых чатов, если таковые появятся) обязаны использовать режим принудительного структурированного вывода. Ответ от AI должен быть строго в формате JSON, соответствующем заранее определенной JSON Schema. Это позволит Backend-сервисам надежно парсить ответ и использовать его в дальнейшей бизнес-логике (создание структуры статьи, генерация мета-тегов) без сбоев из-за неожиданного форматирования.
2. **Алгоритмы RAG (Retrieval-Augmented Generation):** Для Knowledge Engine используется подход RAG. При генерации статьи система не должна полагаться только на внутреннюю память LLM. 
   - Сначала система ищет релевантный контекст (загруженные документы компании, прошлые статьи, tone of voice, семантическое ядро) в векторной базе данных.
   - Найденный контекст подмешивается в system prompt перед отправкой задачи в LLM.
3. **AI Orchestrator Module:** Логика вызова конкретных моделей (Gemini, OpenAI, Anthropic, локальные модели) скрывается за интерфейсами AI Orchestrator. Ни один модуль (например, Content Engine) не должен знать, какая именно модель сейчас используется.

## Последствия
**Положительные:**
- Гарантированная консистентность данных при парсинге ответов от LLM.
- Высокая релевантность и фактическая точность контента благодаря RAG.
- Гибкость в замене провайдеров AI без переписывания бизнес-логики модулей.

**Отрицательные:**
- Требуется дополнительное время и токены на работу со структурированными выводами (некоторые модели могут дольше генерировать JSON).
- Усложнение архитектуры из-за внедрения Vector DB и пайплайнов векторизации данных.
