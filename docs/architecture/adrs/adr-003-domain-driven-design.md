ADR-003
Domain Driven Design
Контекст
Необходимо определить взаимодействие модулей.
Решение
Использовать Domain Driven Design.
Каждый модуль отвечает только за собственную бизнес-область.
Например
Semantic Engine
не знает
как устроен Publisher.
Publisher
не знает
как устроена генерация текста.
Последствия
Запрещены циклические зависимости.
Запрещены общие модели между доменами.
Обмен только через DTO.
